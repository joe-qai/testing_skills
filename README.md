# testing_skills

## 项目简介

testing_skills 是一个专注于软件测试领域的工具集，提供了多种测试相关的功能模块，包括需求分析、测试规划、测试用例生成等。该项目旨在帮助测试工程师和开发团队更高效地进行软件测试工作，提高测试质量和效率。

## 项目结构

```
testing_skills/
├── skill-creator/                 # 技能创建工具
├── doc-based-testcase-generator/  # 基于文档的测试用例生成器
├── requirements-analysis/         # 需求分析工具
├── requirements-analyzer/         # 需求分析专家
├── testcase-generator/            # 测试用例生成器
├── testcase-planner/              # 测试规划器
└── README.md                      # 项目说明文档
```

## 模块功能

### 1. skill-creator

技能创建工具，用于创建、评估和改进技能。包含代理分析、比较和评分功能，支持技能包打包、评估报告生成等。

**核心功能**：
- 技能创建和改进
- 技能评估和评分
- 生成评估报告
- 技能包打包
- 提供可视化评估查看器

### 2. doc-based-testcase-generator

基于需求文档、PRD 或接口文档自动生成结构化测试用例文档。默认采用通用测试用例设计策略（正向、反向、边界值、等价类等）；当用户提到参考 Excel/Word 模板时从 assets 加载模板，提到接口/性能/功能等专用标准时从 references 加载对应说明。适用于从各类文档设计功能、接口、性能及自动化候选用例。

**来源**：通过 skill-creator 技能创建

**核心功能**：
- 支持从 PRD、需求说明、接口文档等中提炼业务规则与接口约束
- 运用系统化的测试设计方法产出高质量测试用例文档
- 支持多种测试类型：功能测试、接口测试、性能测试等
- 可根据用户指定的模板格式输出测试用例

### 2. requirements-analysis

专业需求分析师，通过多轮对话将简短想法转化为详细需求文档。使用此技能当用户需要：(1) 分析和记录需求，(2) 将 EPIC 分解为需求和用户故事，(3) 对多个需求进行优先级排序，(4) 识别利益相关者和依赖关系，(5) 定义验收标准，(6) 创建需求文档。支持单个需求分析、EPIC 分解、MoSCoW/RICE/Kano 等优先级排序方法。

**来源**：来自 skillsmp.com

**核心功能**：
- 对话式需求提取：通过系统化问题将一句话需求转化为全面规格说明
- 分层需求分解：将复杂需求从 EPIC → 需求 → 用户故事进行分解
- 优先级排序：支持 MoSCoW、RICE、价值 vs 成本、Kano、加权评分等方法
- 文档生成：创建结构化的专业需求文档

### 3. requirements-analyzer

专业的需求分析专家，擅长从多种类型文档（Excel、PNG图片、TXT、PDF、Word）中提取和结构化整理需求信息。能够识别功能需求、非功能需求、业务规则、约束条件，并输出标准化的需求分析文档，包括需求依赖关系图和待确认问题清单。

**来源**：通过 skill-creator 技能创建

**核心功能**：
- 支持多种文档格式：Excel、PNG图片、TXT、PDF、Word
- 自动提取功能需求、非功能需求、业务规则、约束条件
- 生成标准化的需求分析文档
- 可视化需求依赖关系图
- 识别并列出待确认问题

### 4. testcase-generator

测试用例生成器 - 同时支持"计划驱动"和"文档驱动"。计划驱动：基于业务流程与用户场景分析、等价类划分、边界值分析、因果图分析等理论，按测试点(POINT)分批生成高质量测试用例，输出为 Markdown 与 Excel。文档驱动：当用户粘贴 PRD/需求/接口文档时，默认运用通用测试设计策略（正向/反向/边界/等价/状态/场景），并可结合 references 与 assets 模板组织输出。

**来源**：经过改造

**核心功能**：
- 基于测试点生成结构化测试用例
- 覆盖核心业务、正向、反向、边界、异常等场景
- 支持 Markdown 和 Excel 格式输出
- 提供用例质量检查和验证
- 支持多种测试类型：功能、接口、性能、安全等

### 5. testcase-planner

测试规划器 - 基于软件测试理论(场景法、等价类划分、边界值分析)将完备需求拆解为结构化的测试规划,输出测试项(ITEM)和测试点(POINT)。

**来源**：来自 skillsmp.com

**核心功能**：
- 识别测试项（ITEM）和测试点（POINT）
- 基于场景法识别独立操作路径
- 评估风险等级（Critical/High/Medium/Low）
- 提炼测试关注点
- 生成结构化测试规划文档

## 使用方法

### 1. 技能创建与管理

使用 `skill-creator` 模块：创建、评估和改进技能，生成评估报告，打包技能包。

### 2. 需求分析

1. 使用 `requirements-analysis` 模块：通过多轮对话将简短想法转化为详细需求文档
2. 使用 `requirements-analyzer` 模块：从多种类型文档中提取和结构化整理需求信息

### 3. 测试规划

使用 `testcase-planner` 模块：将完备需求拆解为结构化的测试规划，输出测试项(ITEM)和测试点(POINT)。

### 4. 测试用例生成

1. 使用 `testcase-generator` 模块：基于测试规划生成结构化测试用例
2. 使用 `doc-based-testcase-generator` 模块：基于需求文档、PRD 或接口文档自动生成测试用例

## 项目特点

- **模块化设计**：各个功能模块相互独立，可根据需要单独使用
- **多种输入格式**：支持从多种文档格式中提取信息
- **标准化输出**：生成结构化、标准化的文档
- **系统化方法**：基于软件测试理论和最佳实践
- **可扩展性**：支持自定义模板和标准

## 应用场景

- **需求分析阶段**：从原始需求中提取和整理结构化需求信息
- **测试规划阶段**：基于需求生成测试规划，识别测试项和测试点
- **测试用例设计阶段**：生成高质量、覆盖全面的测试用例
- **测试执行阶段**：提供标准化的测试用例文档，指导测试执行

## 总结

testing_skills 是一个功能强大的软件测试工具集，通过系统化的方法和标准化的流程，帮助测试工程师和开发团队提高测试效率和质量。无论是需求分析、测试规划还是测试用例生成，都能为用户提供专业、高效的支持。
