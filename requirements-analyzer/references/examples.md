# 需求分析完整示例

## 示例：用户登录与注册需求分析

### 输入
用户提供的原始需求（可能是Word/TXT/PDF/Excel/PNG等格式）：
> 我们系统需要实现用户登录和注册功能。用户可以用手机号或邮箱注册，密码要8位以上。登录错误5次后要锁定账号30分钟。登录成功后跳转首页。

### 完整输出

```markdown
# 用户登录与注册
> 版本：v1.0 | 需求编号：REQ_LOGIN_001 | 创建日期：2025-04-08

---

## 1. 概述

- **需求背景**：现有系统无用户体系，需新增注册登录功能
- **业务目标**：实现用户自主注册和登录，保障账号安全
- **目标用户**：所有终端用户
- **影响范围**：用户模块、权限模块、Session管理

---

## 2. 功能需求

### 2.1 业务流程图

```
开始 → 打开登录页
         ↓
    输入账号密码 → 验证通过？ ─否→ 显示错误提示 → 结束
         ↓是
    登录成功 → 跳转首页 → 结束
```

### 2.2 业务规则表

| 规则ID | 规则描述 | 规则类型 | 优先级 |
|--------|---------|---------|--------|
| BR_001 | 账号支持手机号（11位数字）或邮箱（包含@）格式 | 约束规则 | P0 |
| BR_002 | 密码长度必须 ≥ 8位，且必须包含：大写字母、小写字母、数字、特殊字符之一 | 约束规则 | P0 |
| BR_003 | 连续输入错误密码5次，账号锁定30分钟 | 约束规则 | P0 |
| BR_004 | 登录成功后生成Session，有效期24小时 | 状态规则 | P1 |
| BR_005 | 验证码有效期为5分钟，过期后提示"验证码已过期" | 状态规则 | P1 |

### 2.3 功能点清单

| 功能点ID | 功能点名称 | 功能类型 | 描述 | 优先级 |
|---------|---------|---------|------|--------|
| FP_001 | 账号密码登录 | 新增 | 用户输入正确账号密码后登录 | P0 |
| FP_002 | 验证码登录 | 新增 | 用户输入手机号+短信验证码登录 | P0 |
| FP_003 | 错误锁定 | 新增 | 密码错误5次后锁定账号30分钟 | P0 |
| FP_004 | 用户注册 | 新增 | 新用户填写信息完成注册 | P0 |
| FP_005 | 找回密码 | 新增 | 用户通过验证码找回密码 | P1 |

### 2.4 页面/接口需求

#### 2.4.1 页面元素

| 元素ID | 元素名称 | 元素类型 | 默认值 | 约束条件 | 必填 |
|--------|---------|---------|-------|---------|------|
| E001 | 账号输入框 | input/text | 空 | 11位手机或邮箱格式 | 是 |
| E002 | 密码输入框 | input/password | 空 | 8-20位，含复杂度要求 | 是 |
| E003 | 登录按钮 | button | — | — | — |
| E004 | 错误提示区 | text | 隐藏 | — | — |
| E005 | 验证码输入框 | input/text | 空 | 6位数字 | 是 |
| E006 | 获取验证码按钮 | button | — | 60秒后可重新获取 | — |

#### 2.4.2 接口需求

| 接口ID | 接口名称 | 请求方式 | 关键参数 | 参数约束 | 响应约束 |
|--------|---------|---------|---------|---------|---------|
| API_001 | 登录接口 | POST | username, password | password 长度 8-20 | 返回 token+userId |
| API_002 | 注册接口 | POST | username, password, code | username 需验证唯一性 | 返回 userId |
| API_003 | 获取验证码 | POST | phone | 60秒内只能获取一次 | 返回 code |
| API_004 | 验证码登录 | POST | phone, code | code 6位5分钟内有效 | 返回 token |

---

## 3. 数据需求

### 3.1 数据字段定义

| 字段名 | 数据类型 | 长度限制 | 取值范围 | 默认值 | 备注 |
|-------|---------|---------|---------|-------|------|
| userId | String | 32 | — | 自动生成 | UUID |
| username | String | 50 | 手机号或邮箱 | — | 唯一索引 |
| password | String | 128 | 加密后存储 | — | bcrypt加密 |
| phone | String | 11 | 手机号格式 | — | 可空 |
| email | String | 100 | 邮箱格式 | — | 可空 |
| loginFailCount | Integer | — | 0-5 | 0 | 错误次数 |
| lockUntil | Timestamp | — | — | null | 锁定截止时间 |
| sessionToken | String | 128 | — | — | 登录令牌 |
| sessionExpireAt | Timestamp | — | — | 24小时后 | 过期时间 |

### 3.2 数据状态机

| 状态 | 可转换至 | 触发条件 |
|-----|---------|---------|
| 正常 | 锁定 | 连续错误5次 |
| 锁定 | 正常 | 锁定超时30分钟后自动解锁 |

## 4. 异常与边界需求

### 4.1 异常场景清单

| 异常ID | 异常场景 | 系统行为 | 用户提示信息 |
|-------|---------|---------|-------------|
| EX_001 | 账号为空 | 阻止提交 | 请输入账号 |
| EX_002 | 密码为空 | 阻止提交 | 请输入密码 |
| EX_003 | 账号不存在 | 显示错误 | 账号不存在 |
| EX_004 | 密码错误 | 显示错误 | 密码错误，还剩N次机会 |
| EX_005 | 账号已锁定 | 显示错误 | 账号已锁定，请30分钟后重试 |
| EX_006 | 验证码过期 | 显示错误 | 验证码已过期，请重新获取 |
| EX_007 | 验证码错误 | 显示错误 | 验证码错误 |
| EX_008 | 账号格式错误 | 阻止提交 | 请输入正确的手机号或邮箱格式 |
| EX_009 | 密码复杂度不足 | 阻止提交 | 密码需包含大小写字母、数字和特殊字符 |
| EX_010 | 账号已被注册 | 显示错误 | 该账号已被注册 |

### 4.2 边界值清单

| 边界ID | 字段 | 最小值 | 临界最小 | 正常值 | 临界最大 | 最大值 | 边界类型 |
|-------|------|-------|---------|-------|---------|-------|---------|
| BV_001 | 密码长度 | 0 | 7 | 12 | 9 | 20 | 闭区间 |
| BV_002 | 错误次数 | 0 | 4 | 3 | 6 | 5 | 离散值 |
| BV_003 | 验证码有效期(分钟) | 0 | 4 | 3 | 6 | 5 | 闭区间 |
| BV_004 | 锁定时间(分钟) | 0 | 29 | 30 | 31 | 60 | 闭区间 |

## 6. 验收标准（Acceptance Criteria）

| 验收ID | 验收描述 | 测试方法 | 预期结果 |
|-------|---------|---------|---------|
| AC_001 | 使用正确账号密码能成功登录 | 功能测试 | 跳转首页，显示用户信息 |

## 8. 约束条件

### 技术约束
- 前端框架：React
- 后端框架：Spring Boot
- 数据库：MySQL 8.0
- 缓存：Redis
- 加密：bcrypt

### 业务约束
- 单个用户最多绑定5个账号
- 验证码每日发送上限10条

### 用户体验约束
- 操作步骤不超过3步
- 错误提示需明确易懂
- 登录页面加载不超过2秒
```
