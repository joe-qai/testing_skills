# 需求分析输出格式标准

## 一、必含章节（完整版）

| 章节 | 说明 | 必填 |
|-----|------|------|
| 1. 概述 | 需求名称、版本、需求编号、背景、目标、用户、影响范围 | 是 |
| 2. 功能需求 | 业务流程图、业务规则表、功能点清单、页面/接口需求 | 是 |
| 3. 数据需求 | 数据字段定义、数据状态机 | 视情况 |
| 4. 异常与边界 | 异常场景清单、边界值清单 | 视情况 |
| 5. 非功能需求 | 性能、安全、兼容性等 | 是 |
| 6. 验收标准 | 可测试的验收条件 | 是 |
| 7. 测试设计指导 | 测试策略、覆盖率要求 | 是 |
| 8. 约束条件 | 技术/业务/体验约束 | 视情况 |
| 9. 需求依赖关系图 | 功能依赖关系 | 是 |
| 10. 待确认问题 | 需澄清的问题 | 视情况 |
| 11. 附录 | 术语表/用户角色等 | 视情况 |

## 二、编号规则

| 类型 | 前缀 | 示例 |
|-----|------|------|
| 需求 | REQ_ | REQ_LOGIN_001 |
| 业务规则 | BR_ | BR_001 |
| 功能点 | FP_ | FP_001 |
| 页面元素 | E | E001 |
| 接口 | API_ | API_001 |
| 异常 | EX_ | EX_001 |
| 边界值 | BV_ | BV_001 |
| 验收标准 | AC_ | AC_001 |
| 待确认问题 | Q | Q-001 |

## 三、优先级定义

| 优先级 | 含义 | 说明 |
|-------|------|------|
| P0 | 紧急 | 必须实现，阻塞发布 |
| P1 | 高 | 重要功能，应在首个版本实现 |
| P2 | 中 | 优化功能，可后续迭代 |

## 四、业务规则类型

| 类型 | 说明 |
|-----|------|
| 约束规则 | 限制某种行为（如：密码必须包含XX） |
| 计算规则 | 定义数据如何转换（如：折扣 = 原价 × 0.8） |
| 状态规则 | 定义状态如何流转（如：订单超时 → 自动取消） |
| 权限规则 | 定义谁能做什么（如：只有管理员能删除用户） |

## 五、表格格式标准

### 功能需求表
```
| 功能点ID | 功能点名称 | 功能类型 | 描述 | 优先级 |
|---------|---------|---------|------|--------|
| FP_001 | xxx | 新增/修改/删除 | xxx | P0 |
```

### 业务规则表
```
| 规则ID | 规则描述 | 规则类型 | 优先级 |
|--------|---------|---------|--------|
| BR_001 | xxx | 约束规则 | P0 |
```

### 页面元素表
```
| 元素ID | 元素名称 | 元素类型 | 默认值 | 约束条件 | 必填 |
|--------|---------|---------|-------|---------|------|
| E001 | xxx | input/text | 空 | 4-20字符 | 是 |
```

### 接口需求表
```
| 接口ID | 接口名称 | 请求方式 | 关键参数 | 参数约束 | 响应约束 |
|--------|---------|---------|---------|---------|---------|
| API_001 | xxx | POST | username, password | password≥8位 | 返回token |
```

### 异常场景表
```
| 异常ID | 异常场景 | 系统行为 | 用户提示信息 |
|-------|---------|---------|-------------|
| EX_001 | 用户名为空 | 阻止提交 | 请输入用户名 |
```

### 边界值表
```
| 边界ID | 字段 | 最小值 | 临界最小 | 正常值 | 临界最大 | 最大值 | 边界类型 |
|-------|------|-------|---------|-------|---------|-------|---------|
| BV_001 | 密码长度 | 0 | 7 | 12 | 9 | 20 | 闭区间 |
```

### 验收标准表
```
| 验收ID | 验收描述 | 测试方法 | 预期结果 |
|-------|---------|---------|---------|
| AC_001 | 正确账号密码登录成功 | 功能测试 | 跳转首页 |
```

### 验收标准表
```
| 验收ID | 验收描述 | 测试方法 | 预期结果 |
|-------|---------|---------|---------|
| AC_001 | 正确账号密码登录成功 | 功能测试 | 跳转首页 |
```

## 六、业务流程图格式

```
开始 → 打开登录页
         ↓
    输入账号密码 → 验证通过？ ─否→ 显示错误提示 → 结束
         ↓是
    登录成功 → 跳转首页 → 结束
```

## 七、需求依赖关系图格式

```
[FR-001: 用户登录]
    |
    +-- [FR-002: 订单创建] (依赖: 用户登录)
    |
    +-- [FR-003: 订单查询] (依赖: 用户登录)
```

## 八、数据状态机格式

```
| 状态 | 可转换至 | 触发条件 |
|-----|---------|---------|
| 正常 | 锁定 | 连续错误5次 |
| 锁定 | 正常 | 锁定超时30分钟后自动解 |
```
