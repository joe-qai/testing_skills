# 测试用例文本协议 v0.2

> 版本: v0.2  
> 创建时间: 2026-04-19  
> 状态: 正式使用

## 1. 概述

本协议定义了测试用例的文本格式规范，用于：
- 从 Markdown 文件中解析结构化测试用例
- 验证用例格式是否符合标准
- 检测重复用例

## 2. 格式规范

### 2.1 用例标题

```markdown
## [P0-P3][反向可选] 用例标题
```

**规则**：
- 必须以 `## ` 开头
- 优先级格式为 `[P0]`、`[P1]`、`[P2]` 或 `[P3]`
- `[反向]` 为可选标记，表示反向测试用例
- 标题文本不能为空

**示例**：
```markdown
## [P1] 验证首次登录成功
## [P2][反向] 验证输入错误验证码提示
## [P0] 验证订单支付功能
```

### 2.2 字段定义

用例标题后跟随字段，每个字段以 `[字段名]` 开头：

| 字段 | 是否必填 | 说明 | 示例 |
|------|---------|------|------|
| `[测试类型]` | 是 | 用例类型，见 2.3 | `[测试类型] 功能` |
| `[前置条件]` | 否 | 执行用例前的条件 | `[前置条件] 用户已注册` |
| `[测试步骤]` | 是 | 操作步骤，见 2.4 | `[测试步骤] 1. 打开登录页...` |
| `[预期结果]` | 是 | 期望的输出结果 | `[预期结果] 1. 登录成功...` |
| `[备注]` | 否 | 额外说明 | `[备注] 仅适用于移动端` |

**规则**：
- 字段之间不要空行，保持紧凑格式
- 字段内容可以跨多行（缩进 continuation）
- 字段顺序不强制，但建议按上表顺序

**示例**：
```markdown
## [P1] 验证首次登录成功
[测试类型] 功能
[前置条件] 用户未注册；短信服务正常
[测试步骤] 1. 打开登录页面，输入未注册的11位手机号，点击"获取验证码"。2. 输入收到的验证码，点击"登录"按钮
[预期结果] 1. 收到6位数字验证码短信，按钮变为60秒倒计时。2. 自动创建新账号，登录成功并跳转至首页
```

### 2.3 测试类型

有效的测试类型值：

| 类型 | 说明 |
|------|------|
| 功能 | 功能测试 |
| 兼容性 | 兼容性测试 |
| 易用性 | 易用性测试 |
| 性能 | 性能测试 |
| 稳定性 | 稳定性测试 |
| 安全性 | 安全性测试 |
| 可靠性 | 可靠性测试 |
| 效果（AI类、资源类） | AI/资源效果测试 |
| 效果（硬件器件类） | 硬件效果测试 |
| 可维护性 | 可维护性测试 |
| 可移植性 | 可移植性测试 |
| 埋点 | 埋点测试 |

### 2.4 步骤格式

**推荐格式**：使用 `。` 或 `；` 分隔步骤

```
[测试步骤] 1. 第一步操作。2. 第二步操作。3. 第三步操作
```

**支持的格式**：
1. `1. xxx。2. xxx`（句号分隔）
2. `1. xxx；2. xxx`（分号分隔）
3. `1. xxx\n2. xxx`（换行分隔）

**规则**：
- 步骤编号必须以数字开头，后跟 `. `
- 编号可以不连续（预期结果中常见，如只有第3、6步）
- 步骤内容不能为空

**示例**：
```markdown
[测试步骤] 1. 打开登录页面。2. 输入手机号。3. 点击获取验证码
[预期结果] 1. 页面加载完成。3. 验证码发送成功。6. 按钮变为倒计时状态
```

### 2.5 完整示例

```markdown
## [P1] 验证首次登录成功
[测试类型] 功能
[前置条件] 用户未注册；短信服务正常
[测试步骤] 1. 打开登录页面，输入未注册的11位手机号，点击"获取验证码"。2. 输入收到的验证码，点击"登录"按钮
[预期结果] 1. 收到6位数字验证码短信，按钮变为60秒倒计时。2. 自动创建新账号，登录成功并跳转至首页

## [P2][反向] 验证输入错误验证码提示
[测试类型] 功能
[前置条件] 用户已获取验证码
[测试步骤] 1. 输入错误的6位数字，点击"登录"按钮
[预期结果] 1. 登录失败，停留在登录页，提示"验证码错误，请重新输入"

## [P0] 验证订单支付功能
[测试类型] 功能
[前置条件] 用户已登录，购物车中有商品
[测试步骤] 1. 进入购物车，选择商品。2. 点击"去结算"。3. 选择支付方式。4. 完成支付
[预期结果] 1. 进入订单确认页。2. 进入支付页。3. 支付方式列表展示。4. 支付成功，订单状态更新
[备注] 支持微信支付、支付宝、银行卡
```

## 3. 解析规则

### 3.1 用例边界

- 用例从 `## ` 标题行开始
- 用例到下一个 `## ` 标题行或文件结束结束
- 空行会被跳过

### 3.2 字段解析

- 字段标签格式：`[字段名]`
- 字段内容：标签后的剩余部分
- 多行内容：缩进的后续行会追加到当前字段

### 3.3 步骤解析

1. 先尝试按换行分割（每行 `数字. 内容` 格式）
2. 如果失败，使用正则 `(^|[。；，])\s*(\d+)\.\s*` 匹配步骤编号
3. 步骤内容为从当前编号结束到下一个编号开始

## 4. 验证规则

### 4.1 必填字段

- `[测试类型]` 不能为空
- `[测试步骤]` 不能为空
- `[预期结果]` 不能为空

### 4.2 格式验证

- 标题必须符合 `## [P0-P3][反向] 标题` 格式
- 测试类型必须在有效值列表中
- 步骤和预期结果必须能正确解析

### 4.3 重复检测

使用 TF-IDF + 余弦相似度算法：
- 合并标题、步骤、预期结果为完整文本
- 计算 TF-IDF 向量
- 计算余弦相似度
- 相似度 >= 0.7 判定为疑似重复
